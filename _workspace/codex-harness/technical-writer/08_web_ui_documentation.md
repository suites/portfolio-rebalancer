# Web GUI 문서화 기록

## 범위

- 운영용 Web GUI의 정보 구조, 화면 흐름과 공통 컴포넌트를 정의했다.
- README, SPEC, TODO와 AGENTS의 GUI 관련 계약을 함께 정렬했다.

## 주요 결정

- Web GUI를 주 운영 인터페이스로, CLI를 자동화·진단·복구 보조 인터페이스로 정의한다.
- GUI와 CLI는 동일한 애플리케이션 서비스, 계산기, Risk Gate와 주문 원장을 사용한다.
- 초보 사용자는 GUI만으로 설정, 점검, 계획 검토, Paper 실행과 결과 확인을 완료할 수 있어야 한다.
- 설정 저장, 계획 생성과 주문 제출을 분리하고 상태 불명 주문의 수동 재제출을 금지한다.

## 검토 결과

- 초보 사용자 오류 메시지는 원인, 보호 조치, 다음 행동과 금지 행동을 설명한다.
- Web 보안, 접근성, 반응형, 멱등성 및 감사 로그 요구사항을 구현 백로그와 연결했다.
- 문서의 구현 기준은 `docs/WEB_UI.md`이며 시스템 안전 계약은 `docs/SPEC.md`가 우선한다.
